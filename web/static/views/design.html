Design and Coding Style Guide

Project Design

This project follows standard coding practices that include, but are not limited to, inheritance, abstraction, encapsulation, and polymorphism. Try to keep a design oriented mind when adding new code to the repository. For a full overview of the current project design please see the UML diagram.

The major class is FlashPhotoApp which contains the main handling logic for the program. This class handles all of our filters, tools, and UI interactions. After parsing the information from the UI it then hands off the hard work to the libphoto library.

Each class in the library is either an interface for a toolset (filter/brush), or a class that inherits from these interfaces. Please reference the UML for any questions on the structure of these classes.

Adding a new brush

Adding a new brush is fairly simple. The brushes are built off of a mask structure, which is an MxN matrix, that is applied from the center of the click. This mask is made on creation of the object by a call to new coolTool(color, width, height). In the inherited class coolTool you will then specify a function called FillInfluence. This function will create the MxN matrix, and tell it the values for the intensity the color should be applied at each matrix-relative point. You will then have to add the tool to the UI and the ToolList in FlashPhotoApp. To see how to add it to the UI please see the section "Adding to the UI". Reference any of the preexisting classes that inherit from DrawTool for more information.

Adding a new filter

Much like adding a brush, adding a filter is based off of an inherited class structure. They all inherit from Filter.h. A new filter will need to have a function called ApplyFilter which contains the logic for sweeping through the pixelBuffer and applying logic to each pixel. The logic will vary depending on the filter being implemented. This filter will then need to be added to the FilterFactory, as well as to the UI. To see how to add it to the UI please see the section "Adding to the UI".

Adding to the UI

There are two ways of adding something to the UI. For a brush add the brush to the updateCurrentTool() call back handler. For a filter add it to the gluiControl() callback function. Also be sure to add the filter to the UI_CONTROL_TYPE enum and any brush to the ToolType enum.

Coding Style

There are a few strict rules for coding on this project, outside of that the best design practice is preferred. For any conditional statements (eg, if, for, while) this is the preferred format:

if (statement) {
    //code
} else {
    //code
}

The preferred comment structure should precede every function as follows:

/* Function description
* Parameter description
* Return description */

All variable and function names should be camelCase. For all other practices please use common sense and best practices.
