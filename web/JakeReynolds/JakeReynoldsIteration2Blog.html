My contribution

    My main contributon to this project was the image library code. These libraries were a beast, but I learned a lot through implementing them.

    There were a lot of headaches with the libraries and how to implement them, but I think the largest issues I had was with linking them. The Makefile needs to be able to configure the libraries if they are not installed, so I think my favorite part of this code was figuring out how to do the line "--prefix=actual prefix". I achieved this by using this command in the Makefile. cd $(PNG_PATH); ./configure --prefix=${PWD}/$(PNG_PATH) --enable-shared=no". This uses the current working directory (ignoring the cd at the beginning of this command), and appends the path to the current library. There are better ways of doing this, but I did this without looking anything up, so I'm pretty happy with my solution 

                ImageHandler *loader = new ImageHandler();

                //Pass in height and width by reference so we can resize the window after
                int Height, Width;
                PixelBuffer *newBuffer = loader->loadimage(m_fileName, Height, Width);
                if (newBuffer == NULL) {
                    std::cout << "Error loading image" << std::endl;
                }
                setWindowDimensions(Width, Height);

    Once I finished linking the libraries and making them work I had to dig through the mountains of documentation to find how to deal with these images. The TAs said we are allowed to use code found online and in the documentation for this part, so I won't talk about that code since I mostly customized pieces I found in the docs and online. The API I made for the image handling is pretty nice though. If you want to load an image you will pass it the name of a file, and two uninitialized integers for Height and Width. These integers will be set to the height and width of the new pixel buffer so that we can resize the window dimensions. I tried finding a way to pass in the setWindowsDimensions function to the loadimage call so the last part wouldn't be necessary, but was unable to find a way. 

                ImageHandler *loader = new ImageHandler();

    The other part of the API is the save image functionality. You pass in the filename and the PixelBuffer and the api saves the current buffer for you at the given filename (assuming the filename ends in .jpeg/.jpg or .png. This should be handled by the file explorer though).
